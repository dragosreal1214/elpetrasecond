## Commands

```bash
npm run dev       # start dev server (vite dev)
npm run build     # production build → dist/
npm run preview   # preview the production build locally
npm run lint      # eslint (TypeScript + React Hooks + Prettier)
npm run format    # prettier --write .
```

There are no tests. Both `bun` and `npm` work; `npm` is the canonical runner for Vercel.

## Architecture

**Stack:** Vite 7 · React 19 · TypeScript · TanStack Router · Tailwind CSS v4 · Radix UI

### Routing

File-based routing via `@tanstack/router-plugin`. Each file in `src/routes/` becomes a route:

- `src/routes/__root.tsx` — root layout (Navbar, Footer, WoltFloat wrapped around `<Outlet />`)
- `src/routes/index.tsx` — `/`
- `src/routes/meniu.tsx` — `/meniu`
- `src/routes/galerie.tsx` — `/galerie`
- `src/routes/contact.tsx` — `/contact`

**Never edit `src/routeTree.gen.ts`** — it is auto-regenerated by the Vite plugin on every `dev`/`build`.

To add a route: create `src/routes/new-page.tsx` exporting `const Route = createFileRoute("/new-page")({ component: ... })`. The plugin regenerates the tree automatically.

The router is instantiated in `src/router.tsx` via `getRouter()` and mounted in `src/main.tsx`.

### Styling

Tailwind CSS v4 with `@tailwindcss/vite`. All custom tokens and utilities live in `src/styles.css`:

- **Design tokens** defined as CSS custom properties on `:root` — primary color is `--gold: #E8720E`, background `#0A0A0A`
- **Fonts:** `--font-display: 'Bebas Neue'` (headings, labels, prices), `--font-sans: 'Poppins'` (body)
- **Utility classes** defined in `@layer utilities`: `.btn-gold`, `.btn-gold-outline`, `.card-lift`, `.badge-stat`, `.img-zoom`, `.scroll-badge`, `.menu-leader`, `.section-label-vertical`
- Tailwind scans only `src/` (`@source "../src"` in the CSS file)

Use the custom utility classes rather than duplicating their styles inline.

### Components

- `src/components/` — three layout components used in `__root.tsx`: `Navbar`, `Footer`, `WoltFloat`
- `src/components/ui/` — shadcn/ui Radix primitives (accordion, dialog, sheet, etc.) — add new ones via the shadcn CLI if needed

### Deployment

Deployed to **Vercel** from the `master` branch of `github.com/dragosreal1214/elpetrasecond`. Every `git push` triggers a deploy.

`vercel.json` rewrites all paths to `/index.html` for client-side routing — this is required and must stay.

The `wrangler.jsonc` at the root is a leftover artifact from the original Cloudflare Workers setup; it is not used and can be ignored.
